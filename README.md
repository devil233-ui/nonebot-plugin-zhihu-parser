## 配置项
```
# 单次解析最多输出的图片/视频总量阈值
ZHIHU_MAX_IMAGES = 20
# 合并转发的单批次图片最大数量（推荐 5~10）
ZHIHU_BATCH_SIZE = 5
# 同一个链接的冷却时间（单位：秒。86400秒 = 24小时）
ZHIHU_DEBOUNCE_TIME = 86400
# 自动清理多久以前的缓存文件（单位：秒。604800秒 = 7天）
ZHIHU_CACHE_MAX_AGE = 604800
# ck
ZHIHU_COOKIE=""
```

## 已知问题

目前测试所谓任意风控部分均可通过单独发送解决，其实并不能确定是风控引起的；又，最初测试过一个批次输出50张图都能顺利，可以排除体积原因。

这个奇妙风控在原项目同样存在（但没有备用方案），所以总之目前本项目只能通过独立发送解决这个问题，希望有高人指点。
